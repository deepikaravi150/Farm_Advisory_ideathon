FarmAdvisor - Full App Screen and Widget Input/Process/Output Flow
Generated on: 2026-06-05
Purpose: Human-readable reference explaining every main page and widget from login through the final app screens. Each section says what input the widget takes, where the input comes from, which file/API/data document it refers to, how it processes the data, and what output the farmer sees.

================================================================================
0. GLOBAL RULES USED BY THE WHOLE APP
================================================================================

Authentication source:
- Browser cookie: auth_token
- Created by: app/api/auth/login/route.ts
- Verified by: lib/auth.ts
- Protected by: middleware.ts
- Used by almost every private page and API route to identify farmer_id.

Language source:
- Locale values: en, hi, ta
- Static text source: public/locales/en/common.json, public/locales/hi/common.json, public/locales/ta/common.json
- Runtime language switcher: components/layout/LanguageSwitcher.tsx
- Dynamic translation API: app/api/translate/route.ts
- Output expectation: UI labels, LLM responses, crop plans, soil summaries, current widgets, weather guidance, and chat text should follow selected language.

Main farmer data source:
- DynamoDB table: farmer_profiles
- Helper file: lib/aws/dynamodb.ts
- Profile API: app/api/farmer/profile/route.ts
- Main fields used across app:
  - farmer_id
  - name
  - phone
  - address
  - land_coordinates
  - typography
  - land_area_acres
  - land_picture_s3_key
  - preferred_language
  - memory

Main AI source:
- OpenAI helpers: lib/ai/openai.ts
- RAG helper: lib/ai/rag.ts
- Vector store: lib/ai/vectorstore.ts
- Model env: OPENAI_CHAT_MODEL
- API key env: OPENAI_API_KEY

Main weather source:
- Weather helper: lib/weather.ts
- Weather API: app/api/weather/route.ts
- External API: Open-Meteo
- Reverse geocoding: Nominatim
- Coordinates source: farmer_profiles.land_coordinates

Main storage source:
- DynamoDB tables:
  - farmer_profiles
  - chat_history
  - crop_plans
  - soil_reports
  - government_schemes
- S3 media bucket:
  - S3_BUCKET_MEDIA
  - used for land images, soil PDFs/images, soil extraction JSON, and crop images where uploaded.

1. LOGIN PAGE
================================================================================

Page file:
- app/(auth)/login/page.tsx

Main widgets:
- Language switcher
- Phone input
- Password input with show/hide toggle
- Login button
- Error box
- Register link

1.1 Language Switcher
Input:
- User chooses language: English, Hindi, or Tamil.
Source:
- components/layout/LanguageSwitcher.tsx
- Locale messages in public/locales/*/common.json
Processing:
- Updates app locale cookie/client state.
- Re-renders text through next-intl.
Output:
- Login page labels and placeholders change language.

1.2 Phone Input
Input:
- Farmer enters 10 digit phone number.
Source:
- User typing in login form.
Processing:
- Stored in React state as phone.
- Sent to POST /api/auth/login.
Output:
- Phone becomes the lookup key for farmer account.

1.3 Password Input
Input:
- Farmer enters password.
Source:
- User typing in login form.
Processing:
- Stored in React state as password.
- Show/hide button only changes input type between password and text.
- Sent to POST /api/auth/login.
Output:
- Password is validated server-side against bcrypt hash.

1.4 Login Button
Input:
- phone
- password
Source:
- Login form state.
Processing:
- Calls app/api/auth/login/route.ts.
- API validates input with Zod.
- API looks up farmer by phone using app/api/auth/farmers.ts.
- DynamoDB query uses phone-index if available, with scan fallback.
- bcrypt compares password with password_hash.
- JWT is created by lib/auth.ts.
- auth_token cookie is set.
Output:
- Success: user goes to /dashboard.
- Failure: error message is displayed.

1.5 Error Box
Input:
- API error or network error.
Source:
- Login API response.
Processing:
- React state error is set.
Output:
- Farmer sees invalid credentials, failed login, or network error.

2.1 Step Indicator
Input:

Source:
- Register page React state.
Processing:
- stepIndex decides active/completed step.
Output:
- Farmer sees progress through account creation.

2.2 FarmerDetailsForm
File:
- components/register/FarmerDetailsForm.tsx
Input:
- farmerId
- name
- phone
- address
- typography / land type
- landAreaAcres
Source:
- Farmer form fields.
- Labels and validation messages from next-intl register namespace.
Processing:
- react-hook-form collects fields.
- Zod validates:
  - farmerId required
  - name minimum length
  - phone must be 10 digits
  - address minimum length
  - landAreaAcres positive number
- On success, sends data back to register page through onNext.
Output:
- Valid form data is saved in RegisterPage state.
- UI advances to map step.

2.3 LandMapSelector
File:
- components/register/LandMapSelector.tsx
Input:
- initialAddress from FarmerDetailsForm.
- Farmer search query.
- Farmer clicks on map points.
Source:
- Address from register form.
- Nominatim search API for geocoding.
- Leaflet map click events.
- ESRI satellite and label tile layers.
Processing:
- If address is provided, it tries to geocode address + Tamil Nadu India.
- Pans/zooms map to search result.
- Farmer clicks points to create land boundary.
- Double click or Done completes polygon.
- Requires at least 3 points before moving forward.
- onChange returns coordinates as [{lat,lng}].
Output:
- landCoordinates array is saved in RegisterPage state.
- Farmer sees drawn land polygon on satellite map.

2.4 Password Step
Input:
- password
- confirmPassword
Source:
- Farmer typing.
Processing:
- Client checks password match.
- Client checks password length minimum 8.
- Show/hide toggles input type.
Output:
- Valid password is included in registration payload.

2.5 Create Account Button
Input:
- formData from FarmerDetailsForm
- coords from LandMapSelector
- password
Source:
- Register page state.
Processing:
- Calls POST /api/auth/register.
- API validates body.
- API checks duplicate farmer_id and phone.
- Password is hashed.
- Profile is stored in DynamoDB farmer_profiles.
Output:
- Success: redirects to /login?registered=1.
- Failure: error shown.

4.3 TodayActionBanner
File:
- components/dashboard/TodayActionBanner.tsx
Input:
- current weather
- forecast array
- selected active/latest plan
Source:
- app/dashboard/page.tsx passes data from lib/weather.ts and crop_plans.
- lib/farm-advice.ts provides weatherVerdict and nextStepFromPlan.
Processing:
- If no weather/forecast, hides itself.
- Computes weather verdict from forecast.
- Computes next active/upcoming crop step from plan using current date.
- Weather urgent condition wins over crop task.
- Otherwise active crop task or upcoming task becomes headline.
Output:
- A single "today's plan" banner:
  - urgent weather warning, or
  - crop stage action, or
  - general weather suggestion.
- Link to /crop-plan when plan exists.

4.4 WeatherWidget
File:
- components/dashboard/WeatherWidget.tsx
Input:
- current weather
- forecast array
- selected locale
Source:
- app/dashboard/page.tsx
- lib/weather.ts
- lib/farm-advice.ts
Processing:
- Uses weatherVerdict for overall weekly conclusion.
- Uses currentWeatherFlags for heat/humidity/wind flags.
- Shows only first 7 forecast days.
- Farmer can click a forecast day.
- Day modal calculates field risk:
  - High for thunderstorm, heavy rain, or high heat.
  - Medium for moderate rain/drizzle/heat.
  - Low otherwise.
- Day guidance is generated from description/rain/temp.
Output:
- Current weather hero card.
- Today's action recommendation.
- 7-day forecast strip.
- Detailed overlay for selected day with temperature, rainfall, field risk, and guidance.

4.10 ChatPanel
File:
- components/chatbot/ChatPanel.tsx
Input:
- User typed question.
- Optional crop image.
- Optional voice transcript.
- Selected locale.
- Last 6 local messages as history.
Source:
- User input UI.
- VoiceInput component.
- Camera/file input.

Processing:

6.2 SoilReportUpload
File:
- components/profile/SoilReportUpload.tsx
Input:
- PDF/image soil report file.
- selected locale.
Source:
- Farmer file upload.
- useDropzone accepts image/* and PDF.
- Max client file size: 10 MB.
Processing:
- Sends multipart form to POST /api/soil/upload with file and locale.
- API:
  - authenticates farmer
  - computes file hash
  - checks duplicate soil report for same farmer
  - uploads original file to S3 if new
  - extracts soil values with OpenAI
  - uploads extracted JSON to S3
  - stores normalized result in DynamoDB soil_reports
  - marks report as current
- Component then calls /api/translate with kind soil_report whenever locale/result changes.
- Translation preserves technical values and translates farmer-facing text.
Output:
- Dropzone/analyzing UI.
- Error box if upload/extraction fails.
- Soil result card:
  - extracted title
  - lab name if present
  - what this means
  - key findings
  - pH
  - nitrogen
  - phosphorus
  - potassium
  - AI recommendations

7.3 Crop Plan Generation
API:
- POST /api/crop-plan
Input:
- farmerState
- cropName
- startDate/current crop info
- assessment answers
- locale
Source:
- StateAssessmentModal.
- Farmer profile from DynamoDB.
- Current soil report from DynamoDB.
- Farmer memory from profile.
- Weather forecast from Open-Meteo.
Processing:
- API builds soil context:
  - pH, EC, organic carbon, N/P/K, micronutrients, recommendations.
- API builds profile context:
  - address, land area, land type, coordinates.
- API builds assessment context.
- API builds weather forecast prompt.
- OpenAI generates structured crop plan JSON.
- Plan is normalized.
- Weather warnings are added to milestones.
- Plan is stored in DynamoDB crop_plans.
Output:
- CropPlan object:
  - cropName
  - startDate
  - milestones
  - totalBudgetEstimate
  - harvestDate
  - sellWindow
  - storageNotes
  - weatherAlerts

8. CHAT API PROCESSING DETAILS
================================================================================

API file:
- app/api/chat/route.ts

Input:
- message
- locale
- history
- optional crop image file

Sources used inside API:
- farmer profile from DynamoDB farmer_profiles
- latest soil report from DynamoDB soil_reports
